# Synthetic Data Generation Guidelines (Safe)

## Goals
- Increase reasoning diversity
- Avoid memorization
- Ensure verifiability

## Allowed Domains
- Math with exact answers
- Coding with unit tests
- Logic puzzles with ground truth

## Forbidden Domains
- Exploit development
- Malware payloads
- Unauthorized access instructions

## Quality Filters
- Deduplicate prompts
- Enforce strict verifier success criteria
- Reject ambiguous tasks
